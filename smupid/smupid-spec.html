smupid spec 1

smupid operates on an array of bytes. In the C interpreter it is an unsigned char[65536] using dynamic memory.

Similar to Brainf__k, the pointer controls where you are reading/writing to in memory. You can only edit memory by adding/subtracting to it or by getting user input.

In the C interpreter, the pointer is an unsigned short.

opcode table

name usage description
reset pointer p set pointer to zero.
add to pointer d[x] add x to pointer.*
add to memory a[x] add x to memory.*
input i set memory to keyboard input.
output o output memory.
loop =[b] { … } if memory == b, all code within {} is skipped. If it doesn’t, that code will loop until it does.**
* x can be positive or negative.
** b can only be a valid unsigned byte value (0-255)
